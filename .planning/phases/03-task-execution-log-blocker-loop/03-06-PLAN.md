# Plan 06 — UAT Retest Control Reliability Fixes

<objective>
Close the second Phase 3 UAT retest gaps by replacing unreliable drawer controls and form-submit paths with visible, dependable task controls: native date/number inputs, explicit click handlers, richer subtask rows, clear participant/date/card display, and understandable code-evidence copy.
</objective>

<scope>
Frontend reliability and clarity only. Do not add Phase 4 acceptance review, notifications, reports, real Git sync, or AI review.
</scope>

<tasks>

<task id="03-06-01" title="Make create and edit forms use visible native time controls and explicit submit buttons">
<read_first>
- `frontend/src/components/task/CreateTaskDrawer.vue`
- `frontend/src/components/task/TaskDrawer.vue`
- `frontend/src/views/ProjectBoardView.vue`
- `frontend/src/types/task.ts`
</read_first>
<action>
- Replace Ant Design `a-date-picker` usage in task create/edit forms with visible native `<input type="date">` controls.
- Add `start_date` as UI-only optional input if backend does not persist it yet; clearly label it as display/planning metadata only, or defer it with a visible note if not stored.
- Keep `due_date` persisted through existing backend field.
- Replace `@finish`-only save/create handling with explicit button `@click` handlers so clicks always call the save/create functions.
- Show an inline success/error message immediately after create/save attempts.
</action>
<acceptance_criteria>
- Create form visibly shows start date and due date controls.
- Confirm create button visibly responds and either creates the task or shows a useful error.
- Edit save button visibly responds and either saves the task or shows a useful error.
- Due date is persisted and appears on the task card after save/create.
</acceptance_criteria>
</task>

<task id="03-06-02" title="Improve task card metadata for participants and dates">

- `frontend/src/components/task/TaskCard.vue`
- `frontend/src/types/task.ts`
- `frontend/src/views/task-execution.spec.ts`

<action>
- Show Owner and participant display names, not just participant counts.

</acceptance_criteria>
</task>
