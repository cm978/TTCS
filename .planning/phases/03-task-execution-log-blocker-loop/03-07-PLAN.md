# Plan 07 — Team Project Navigation Context

<objective>
Repair the Phase 2 to Phase 3 information architecture so users move through a clear context path: team list, team detail, team projects, project detail, then task board.
</objective>

<scope>
Use existing backend team/project APIs. Do not add reporting, notification center, acceptance review, AI review, or real Git integration.
</scope>

<tasks>

<task id="03-07-01" title="Add team detail route and page">
<read_first>
- `frontend/src/router/index.ts`
- `frontend/src/views/TeamProjectStartView.vue`
- `frontend/src/stores/team.ts`
- `frontend/src/api/teams.ts`
- `frontend/src/api/projects.ts`
</read_first>
<action>
- Add `TeamDetailView.vue` and route `/teams/:teamId`.
- Load the selected team, team members, and projects in that team.
- Show team name, description, member count, and project list.
- Provide clear actions for `管理成员`, `创建项目`, and opening a project detail page.
- Change team list links so `查看详情` opens the team detail page before project selection.
</action>
<acceptance_criteria>
- User can open a team detail page from the team list.
- Team detail shows members and projects for that team.
- Team detail is the natural entry to the team's project list.
</acceptance_criteria>
</task>

<task id="03-07-02" title="Add project detail route before task board">
<read_first>
- `frontend/src/router/index.ts`
- `frontend/src/stores/project.ts`
- `frontend/src/types/project.ts`
- `frontend/src/views/ProjectBoardView.vue`
</read_first>
<action>
- Add `ProjectDetailView.vue` and route `/projects/:projectId`.
- Load the project board payload to reuse project, members, and columns without a new backend endpoint.
- Show project name, description, start/end dates, members, and board-column summary.
- Provide explicit actions: `进入任务看板`, `管理成员`, and `返回团队`.
- Change project links from direct board navigation to project detail navigation.
</action>
<acceptance_criteria>
- User sees project detail before entering the task board.
- The task board remains reachable through a deliberate `进入任务看板` action.
- Project detail preserves Phase 2/3 context without adding later-phase reporting.
</acceptance_criteria>
</task>

<task id="03-07-03" title="Make task board context breadcrumbs explicit">
<read_first>
- `frontend/src/views/ProjectBoardView.vue`
- `frontend/src/layouts/AppLayout.vue`
- `frontend/src/stores/team.ts`
- `frontend/src/stores/project.ts`
</read_first>
<action>
- Update task board header to show `团队 / 项目 / 任务看板` context where available.
- Add return links to project detail and team detail.
- Ensure sidebar navigation targets team/project detail when available instead of jumping ambiguously.
</action>
<acceptance_criteria>
- Task board tells the user which project they are working in.
- User can move back to project detail and team detail.
- Sidebar does not create ambiguous jumps into the wrong project.
</acceptance_criteria>
</task>
