# Plan 04 — Frontend Management and Board UI

<objective>
Build the visible Phase 2 collaboration UI: team creation/invitation/member management, project creation, project board with real empty columns, and project member drawer.
</objective>

<user_story>
As a TTCS team administrator or project manager, I want to manage team/project membership and enter an empty but real project board, so that the collaboration workspace is ready for task execution in the next phase.
</user_story>

<threat_model>
T-02-UI-AUTHZ: UI could present actions the current user cannot perform. Hide or disable member-management actions when role data says the user lacks authority.
T-02-UI-DESTRUCTIVE: Removing members or cancelling invitations could happen accidentally. Use confirmation copy for destructive actions.
T-02-UI-FAKE: Fake task cards could misrepresent Phase 2 completeness. Board columns must show empty states only.
T-02-UI-A11Y: Dense management UI can become inaccessible. Maintain labels, focus rings, keyboard order, and 44px action targets.
</threat_model>

<must_haves>
- D-19, D-24: Project creation and board workspace behavior.
- D-25..D-30: Fixed, real, empty default board columns.
- D-32..D-40: Team/project member management UX and role-guard feedback.
</must_haves>
