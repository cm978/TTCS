# Plan 01 — Walking Skeleton Scaffold and Runtime

<objective>
Create the runnable TTCS full-stack skeleton: local MySQL/Redis services, FastAPI `/api/v1` health route, Alembic foundation, Vue 3/Vite/TypeScript app, and shared environment examples.
</objective>

<user_story>
As a developer, I want to start the TTCS backend, frontend, database, and cache locally, so that later auth and product slices have a real full-stack foundation.
</user_story>

<threat_model>
T-01-ENV: Secrets accidentally committed through local configuration. Use `.env.example` with placeholders only and load real values from environment.
T-01-CORS: Over-broad CORS settings allow unexpected origins. Default to local frontend origin and document changes explicitly.
T-01-DB: Migration drift blocks later phases. Introduce Alembic before auth tables land.
</threat_model>

<must_haves>
- D-10: Use the LLD-aligned `backend/` and `frontend/` directory structure.
- D-11: Backend foundation uses FastAPI and SQLAlchemy.
- D-12: Frontend foundation uses Vue 3, Vite, and TypeScript.
- D-13: Introduce Alembic in Phase 1.
- D-14: Provide `docker-compose.yml` for local MySQL 8 and Redis 7.
- D-15: Provide environment examples for database, Redis, JWT, CORS, and frontend API settings.
- D-25: Frontend design must follow UI/UX Pro Max principles.
- D-26: `design-system/MASTER.md` is the frontend visual source of truth.
- D-28: Do not use emoji as structural icons; prefer `lucide-vue-next`.
- D-30: Ant Design Vue must be themed or wrapped with TTCS tokens.
</must_haves>
