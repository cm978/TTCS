# 需求变更说明 - 任务进度追踪增强

**变更编号：** CR-2026-001  
**变更日期：** 2026-03-15  
**变更类型：** 功能增强  
**影响范围：** 需求分析、设计文档、数据库、API

---

## 一、变更背景

### 1.1 问题描述

原设计存在以下不足：
1. **任务中间过程无法追踪** - 只有简单的状态流转，无法了解开发中的具体进度
2. **多人协作支持不足** - 一个任务只能有一个负责人，无法反映实际协作场景
3. **阻塞问题发现滞后** - 缺乏主动阻塞上报机制，问题容易延误

### 1.2 变更目标

1. 增加**工作日志**功能，成员可记录每日工作内容和进度
2. 支持**多人协作**，任务可有多个参与者（上限 5 人）
3. 建立**阻塞通知机制**，快速识别和响应阻塞问题
4. 为二期**Git 集成**预留数据字段

## 三、核心设计决策

### 3.1 任务负责人模型

```
变更前后对比：

变更前：
Task {
  assigneeId: string    // 单个负责人
}

变更后：
Task {
  ownerId: string,      // 任务创建者（不变）
  assigneeIds: string[] // 参与者数组（最多 5 人）
}
```

**设计理由：**
- `ownerId` 确保任务责任明确，始终有人负责
- `assigneeIds` 支持灵活协作，适应实际开发场景

---

### 3.2 进度计算逻辑

```typescript
// 采用客观计算，避免主观偏差
function calculateProgress(task: Task): number {
  const subtasks = getSubtasks(task.id);
  if (subtasks.length === 0) return 0;
  
  const completed = subtasks.filter(s => s.completed).length;
  return Math.round((completed / subtasks.length) * 100);
}
```

**设计理由：**
- 完全基于子任务完成度（100% 客观）
- 避免成员自报进度的主观性
- 激励团队拆分任务为可衡量的子任务

---

### 3.3 阻塞通知机制

```
阻塞上报流程：

成员标记阻塞
    ↓
系统验证阻塞原因（≥10 字符）
    ↓
┌───────────────────────────────────────┐
│ 同时触发：                            │
│ 1. 发送即时通知给任务 Owner           │
│ 2. 看板任务卡片显示红色阻塞标记       │
│ 3. 加入每日站会阻塞报告               │
└───────────────────────────────────────┘
```

**设计理由：**
- 三重保障确保阻塞问题不被遗漏
- 最小 10 字符要求促使成员详细描述问题
- 站会汇总帮助团队快速识别瓶颈

---

### 3.4 Git 集成预留

**work_logs 表预留字段：**
```sql
commit_hash     VARCHAR(40)     -- Git Commit Hash
branch_name     VARCHAR(255)    -- 分支名称
repository_url  VARCHAR(500)    -- 仓库地址
git_synced      BOOLEAN         -- 是否 Git 自动同步
```

**设计理由：**
- 一期使用手动记录，快速上线
- 二期通过 Git Webhook 自动创建日志
- 字段预留避免后期数据库大改

## 五、二期 Git 集成规划

### 5.1 集成方式

```
Git Webhook → TTCS Webhook 接收器 → 解析 Commit 消息 → 自动创建工作日志

Commit 消息格式：
"feat: 实现登录功能 #TASK-123"
                        ↓
                   提取任务 ID
```

### 5.2 自动关联规则

| Commit 消息格式 | 关联行为 |
|---------------|---------|
| `#TASK-123` | 关联到任务#123 |
| `Fixes #123` | 关联并自动完成任务 |
| `WIP #123` | 关联并标记为进行中 |

### 5.3 预留字段使用

| 字段 | 用途 |
|------|------|
| commit_hash | 存储 Commit Hash，支持点击跳转 |
| branch_name | 记录开发分支，便于追溯 |
| repository_url | 多仓库场景下标识来源 |
| git_synced | 区分手动/自动创建的日志 |
