## 3. 用例详细说明

### 3.1 用户管理用例

#### UC-01 用户注册

| 属性 | 描述 |
|------|------|
| **用例 ID** | UC-01 |
| **用例名称** | 用户注册 |
| **主要参与者** | 访客 (ACT-01) |
| **优先级** | 高 |
| **前置条件** | 用户未注册账号 |
| **后置条件** | 用户账号创建成功，状态为待验证 |

**基本流程：**

| 步骤 | 参与者动作 | 系统响应 |
|------|-----------|---------|
| 1 | 访问注册页面 | 显示注册表单 |
| 2 | 输入邮箱、密码、姓名 | 实时验证输入格式 |
| 3 | 点击"发送验证码" | 发送验证邮件 |
| 4 | 输入验证码 | 验证验证码有效性 |
| 5 | 勾选服务协议，点击"注册" | 创建账号，跳转登录页 |

**备选流程：**

- A1：邮箱已被注册
  - 步骤 2 后，系统提示"该邮箱已注册"
  - 用户可使用"找回密码"功能或更换邮箱

- A2：验证码错误
  - 步骤 4 后，系统提示"验证码错误"
  - 用户可重新获取验证码

- A3：验证码超时
  - 步骤 4 后，系统提示"验证码已过期"
  - 用户可重新获取验证码

**业务规则：**

- BR-UC-01：密码长度 8-20 位，必须包含字母和数字
- BR-UC-02：验证码有效期 30 分钟
- BR-UC-03：同一邮箱 1 分钟内只能请求 1 次验证码

**异常处理：**

- E1：邮件发送失败
  - 系统提示"邮件发送失败，请稍后重试"
  - 记录错误日志

**界面原型：**

```
┌────────────────────────────────────┐
│         团队任务协作管理系统         │
├────────────────────────────────────┤
│                                    │
│           创建账号                  │
│                                    │
│  邮箱：[____________________]      │
│  姓名：[____________________]      │
│  密码：[____________________]      │
│  确认：[____________________]      │
│                                    │
│  [发送验证码] 验证码：[______]     │
│                                    │
│  ☐ 我已阅读并同意《用户协议》       │
│                                    │
│        [    注册    ]              │
│                                    │
│  已有账号？[立即登录]               │
└────────────────────────────────────┘
```

#### UC-02 用户登录

| 属性 | 描述 |
|------|------|
| **用例 ID** | UC-02 |
| **用例名称** | 用户登录 |
| **主要参与者** | 访客 (ACT-01) |
| **优先级** | 高 |
| **前置条件** | 用户已注册并验证 |
| **后置条件** | 用户登录成功，建立会话 |

**基本流程：**

| 步骤 | 参与者动作 | 系统响应 |
|------|-----------|---------|
| 1 | 访问登录页面 | 显示登录表单 |
| 2 | 输入邮箱和密码 | - |
| 3 | 可选：勾选"记住我" | - |
| 4 | 点击"登录" | 验证凭证，跳转首页 |

**备选流程：**

- A1：密码错误
  - 步骤 4 后，系统提示"邮箱或密码错误"
  - 记录失败次数

- A2：账号被锁定
  - 步骤 4 后，系统提示"账号已锁定，请 30 分钟后再试"
  - 显示解锁倒计时

- A3：账号未验证
  - 步骤 4 后，系统提示"请先验证邮箱"
  - 提供"重新发送验证邮件"选项

**业务规则：**

- BR-UC-04：连续 5 次登录失败锁定账号 30 分钟
- BR-UC-05：Token 有效期 24 小时
- BR-UC-06："记住我"保持登录 7 天

**异常处理：**

- E1：系统维护中
  - 显示维护公告页面

  - 系统更新参与者列表

#### UC-16 记录工作日志

| 属性 | 描述 |
|------|------|
| **用例 ID** | UC-16 |
| **用例名称** | 记录工作日志 |
| **主要参与者** | 任务参与者 (ACT-05) |
| **优先级** | 高 |
| **前置条件** | 用户是任务参与者 |
| **后置条件** | 工作日志创建，若阻塞则触发通知 |

**基本流程：**

| 步骤 | 参与者动作 | 系统响应 |
|------|-----------|---------|
| 1 | 打开任务详情 | 显示任务信息和工作日志区域 |
| 2 | 点击"记录工作" | 显示工作日志表单 |
| 3 | 输入工时数、工作日期 | 验证工时范围（0.5-24 小时） |
| 4 | 输入工作内容描述 | 支持 Markdown 格式预览 |
| 5 | 可选：勾选"遇到阻塞" | 显示阻塞原因输入框 |
| 6 | 填写阻塞原因 | 最小 10 字符验证 |
| 7 | 点击"保存" | 创建工作日志 |
| 8 | 若标记阻塞 | 发送通知给任务负责人 |

**备选流程：**

- A1：标记阻塞但未填写原因
  - 步骤 7 后，系统提示"请填写阻塞原因"
  - 操作被阻止

- A2：记录过去 30 天以外的日期
  - 步骤 3 后，系统提示"只能记录过去 30 天的工作日志"
  - 日期选择器禁用超出范围的日期

**后置流程：**

- 若日志标记为阻塞：
  1. 系统发送即时通知给任务负责人（Owner）
  2. 任务在看板上显示红色阻塞标记
  3. 自动加入每日站会阻塞报告
